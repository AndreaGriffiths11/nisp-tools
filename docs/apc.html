Appendix C. The NC3TA standards DTD

As previously mentioned, the standards database is implemented as an XML document. The design of the database reflects the original structure of chapter 2 in volume 3. In order to create consistency across the different volumes, the database have been extented to be able to describe mandatory- and emerging standards selected in the NCSP and justified in the traceability matrix included in the rationale document.

In the following, the syntax of the database is described is detail and when appropiate examples of the ??. listing contains a numberd copy of the definition of the DTD used to describe the standard database. Following this listing is a detailed description of the elements used to define the DTD.


<!ELEMENT ta-standards (servicearea+, organisations)>

<!ELEMENT servicearea (serviceclass+)>

<!ATTLIST servicearea
          id      ID       #REQUIRED
          title   CDATA    #REQUIRED>

<!ELEMENT serviceclass (ncsp-list?, (standardrecord | profilerecord)*)>

<!ATTLIST serviceclass
          cid     ID       #REQUIRED
          title   CDATA    #IMPLIED>

  • Line 1: the root element ta-standards contains one or more servicearea elements, and an organisations. The later is used as a lookup table, where the mapping between organisations represented by entities and the text used when formatting standard entries in defined.

  • Line 3: A service area contains of one or more service classes

  • Line 5-7 and 12-14: Both service areas and service classes must have both an id- and a tag attribute

  • Line 10: A serviceclass consist of a possible ncsp-list element and zero or more standardrecoed or profilerecord elements. A ncsp-list is a list, which represents the manatory- and emerging standards. This structure is very similar to the standard class tables in the NCSP.



<!ELEMENT standardrecord (debug?, ((standard?, applicability?, status?) |
                          sremark))>

<!ATTLIST standardrecord
          tag CDATA #REQUIRED
          id ID #IMPLIED>

  • A standardrecord consists of a possible debug element, which is not used for anything but keeping notes of potential problems in a record. This is maybe followed by the elemnts standard, applicability and status or a sremark element.

  • A standardrecord must have an tag attribute and may have an id attribute
