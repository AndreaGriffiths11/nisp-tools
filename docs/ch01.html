Chapter 1. Installation and Configuration

Working with DocBook XML files, requires quite a bit of software. However, the number of packages necessary, depends on the requirements of the user. To simplify things, most of the tools have been bundled in the nisp-tools package. In order e.g. to generate a HTML version of the NATO Interoperability Standards and Profiles, the users only have to install Java, the nisp tools distribution and the sources for the XML version of the NATO Interoperability Standards and Profiles. A detailed description of all the different tools included in this package can be found in Appendix A, Inside the NISP tools distribution.

All software packages can in principle be installed anywhere, but everything should work without a glitch, if some of the packages are installed in a couple of predefined places.

File paths in archives

The user should be aware, that some distributions are archived inside a sub directory, and some are not., e.g. the NISP tools distribution is unpacked into the sub directory nisp-tools-4.1+, where all the files in the NISP tools distribution are located. It is up to the user to ensure this issue is resolved properly. So if the installation instructions recommends that the NISP package should be installed under Windows in the directory c:\nisp-tools, the path to the README file from the NISP tools distribution, should be the file path c:\nisp-tools\README and not the file path c:\nisp-tools\nc3ta-tools-4.1+\README.

1.1. Installation of Java Runtime Environment 1.6.0_22

The installation of the Java 1.6.0_22 is straight forward. The Java 2 Software Development Kit or the Java 2 Run-time environment comes as an archived executable, and is by default installed under windows at c:\jdk1.6.0_22. If the user prefers to use another version of java, please consult the installation instruction of the programs Xerces, Saxon, Batik, Fop, Sun catalog resolver and Ant to identify the system requirements for these tools.[1]

The user must always add the directory, where Java executable are located to the PATH environment variable. Typical values for PATH are:

Java 2 Software Development Kit

   c:\jdk1.6.0_22\bin            (Win32)

   /usr/java/jdk1.6.0_22/bin     (Unix)

Java 2 Runtime Environment

   c:\Program Files\JavaSoft\JRE\1.6.0_22\bin     (Win32)

   /usr/java/jdk1.6.0_22/bin                      (Unix)

The user should also set the environment variable JAVA_HOME. Although normally not required by Java 1.6.0_22, the Ant program requires this variable set for reasons of backward compatibility. Typical values for JAVA_HOME are:

Java 2 Software Development Kit

   c:\jdk1.6.0_22            (Win32)

   /usr/java/jdk1.6.0_22     (Unix)

Java 2 Runtime Environment

   c:\Program Files\JavaSoft\JRE\1.6.0_22     (Win32)

   /usr/java/jdk1.6.0_22                      (Unix)

[1] Note, that the tools have not been tested with Java 2 SDK 1.4, so you are entirely on your own, if you choose this platform.
