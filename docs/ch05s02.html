5.2. Used DocBook Elements

All-though the DocBook XML DTD consists of almost 400 elements. The XML implementation of the NATO Interoperability Standards and Profiles uses only a very limited subset of all the elements defined in the DTD as illustrated in the following table[4], which is based on Version 5.0 of the XML NATO Interoperability Standards and Profiles (dated December 15, 2003). The table clearly illustrates, that the elements mostly used, are actually related to table elements, and the paragraph element para.

The elements used in the NATO Interoperability Standards and Profiles is as described above only a small subset of all the elements in the DocBook DTD. In [WaMu05], a breakdown of the logical stucture of DocBook elements is described.

Any DTD describes a number of elements and the relationship between these documents, however there is no syntactic construct, which identify a single element at the default root elements. Often a written description of the DTD identify a such a root element, however this is not the case for the docbook DTD. Theoretically any element can be a root element in DocBook, but in practice the root element of most DocBook documents are either the article or the book element.

All the documents in the NATO Interoperability Standards and Profiles do have the book element as its root element.
