4.3. DTDs

A Document Type Definition (DTD) describes the legal structure of a document and is an integral part of the XML standard. The DTD describes the formal syntax of a given document and describes precisely, which elements and attributes are allowed, and the context in which the elements are allowed. The DocBook DTD can make statements, such as a sect2 element must be contained inside a sect1 element, Or a row element are only allowed to contain one or more entry or entrytbl elements. It is not a requirement, that the DTD is available when writing XML documents, but whenever people are involved in the process, it is always a good idea, to use a DTD in order to ensure that the XML document is legal. The source distribution comes with a number of DTDs, which are described futher in Section 7.1.4, “DTD”.
